Frames originated as a technology used for knowledge representation in artificial intelligence. They are very close to objects and class hierarchies in object-oriented languages although their fundamental design goals are different. Frames are focused on the explicit and intuitive representation of knowledge whereas objects focus on encapsulation and information hiding.

Original Marvin Minsky's concept lacks some principal features for software design, so I must extend it with ability to store sequential collections. In result extended frame model able to natively represent attribute grammar and AST trees treated as a required model for source code representation, and compilers design.

In practice, the techniques and capabilities of the frame model and object-oriented languages overlap significantly so much as we can treat frames not only a native superset of OOP but they drastically extend object design concepts wider: we can represent any knowledge in frames, and use any programming paradigms as we desire.

It should be especially noted that the frame model includes executable semantic: slots not only be able to be computational, but there are daemons acts as separated computational agents. As we have both knowledge representation and computational data structures, we must go one step further and treat [[frame graph]] systems as the [[homoiconic]] programming language and full powered computational model.

Due to frame model flexibility, we can potentially implement angry type system checkers available in modern functional programming languages. The formal semantics of frame languages gives them a new type of automated reasoning capability known as the classifier. It is an engine that analyzes the various declarations: the definition of types, objects, sets, subsets, relations, etc. The classifier can then automatically deduce various additional relations and can detect when some parts of a model are inconsistent with each other. In this way, many of the tasks that would normally be executed by forward or backward chaining in an inference engine can instead be performed by the classifier.
