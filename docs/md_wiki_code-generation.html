Maybe it's time to move out from standard [[FORTH]] to the integration with mainstream systems? I'm experimenting with an interpreter written in Python, which represents programs in graphs of homoiconic objects, and such alike implementation of FORTH-like language [[metaL]] works reasonably easy not only to use but to implement from a scratch. I throw out FORTH standards, use symbolical addressing in place of ugly memory image integer addresses, and it gives me real freedom to integrate with any system and any modern mainstream object language. For example, last week I was bitten by Scala, and I took the same frame object model in the role of learning sample.

If you need extra processing power, I'm thinking about a method to produce very compact and optimized code able to run on a microcontroller with 2K of RAM, by programming in any high-level language as instrumentation say for example Python, Scala +JVM etc, any huge and resource hungry programming system works up to superclusters.

I mean code generation as the main method and metaprogramming in a whole: write code in superpower languages with rich OOP and ideology you are comfortable with (including Haskel), which generates very small and fast low-level code (embedded C, LLVM, maybe Java for enterprise systems). WIth all OOP, logic inference, functional programming, angry type checking, etc.

For my sorrow, I can't show anything that looks working, but this idea crashes my brain last year, and I can't forget about it despite the lack of any progress. Look on Marvin Minsky frame model as a representation analog of an attribute grammar, add some Prolog-like inference, and you'll feel the idea.
